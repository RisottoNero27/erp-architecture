# Кастомная ERP-система для E-commerce (Архитектура и Системный Анализ)

Проект по проектированию и разработке полноценной ERP-системы для розничной компании, масштабирующей продажи через маркетплейсы. Система заменила децентрализованный учет в Google Sheets, обеспечив строгую консистентность данных и автоматизацию рутины.

## Бизнес-контекст и результаты
*   **Проблема:** Существующая архитектура на базе электронных таблиц не выдерживала роста компании (с 300 до 1000+ заказов в сутки). Возникали ошибки пересорта, рассинхронизация остатков и критические задержки в закупках.
*   **Решение:** Спроектирована и внедрена ERP-система, покрывающая модули PIM (управление каталогом), WMS (склад), SCM (закупки) и интеграцию с Kaspi API.
*   **Результат:** 
    * Пропускная способность увеличена до 1000 заказов в сутки.
    * SLA маркетплейса (процент отмен) снижен с 0.9% до 0.3%.
    * Время складской приемки партий от 2500 единиц сокращено на 85% за счет внедрения Telegram Web App модулей.

## Технологический стек
*   **Архитектура и анализ:** C4 Model, BPMN, OpenAPI (Swagger), Event Sourcing.
*   **Backend:** Python 3.10+, FastAPI, PostgreSQL, SQLAlchemy, Celery, Redis.
*   **Frontend & UI:** React, Telegram Web App (TWA).
*   **CI/CD:** GitHub Actions.
